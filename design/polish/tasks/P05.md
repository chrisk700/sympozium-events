## Findings

- `helm lint --strict` passes clean (only INFO for missing chart icon, not a warning).
- `helm unittest` 26 tests / 4 suites — all pass.
- Values paths in test files are relative to the test file, not the chart root (`../ci/two-routes.yaml`).
- `notContainsDocument` is **not** a valid assertion type in helm-unittest v1.0.3 — causes a panic in ParseTestSuiteFile. Use `hasDocuments: count: N` instead.
- Subchart templates (argo-events bundled as `.tgz`) cannot be referenced by path in helm-unittest. `byo_test.yaml` verifies chart-owned resources still render in BYO mode rather than asserting subchart absence directly.
