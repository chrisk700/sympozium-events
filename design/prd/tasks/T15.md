Full chart install and end-to-end event flow.

## Sub-items

- [x] `helm install sympozium-events ./sympozium-events -f test-values.yaml` with a single webhook route targeting a test `SympoziumInstance`
- [x] POST a test payload to the EventSource webhook endpoint
- [x] Confirm `AgentRun` CR created in `sympozium-system` with correct `instanceRef`, `sessionKey`, and rendered `task`
- [x] Confirm Sympozium controller executes the run to `Succeeded`
- [x] POST the same payload again — **finding**: duplicate AgentRun IS created. Argo Events fires once per HTTP POST regardless of payload identity. Both runs share the same `sessionKey` (JSON of request body). Argo Events NATS dedup window (300s) only deduplicates messages with the same `Nats-Msg-Id` header, not by content. Content-based dedup requires application-level or Sensor filter logic. Document as known limitation; address in T16 if relevant.
