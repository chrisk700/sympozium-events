# P02 — Chart.yaml: version floor annotation

`Chart.yaml` pins `argo-events: 2.4.x` as a dependency but does not surface the minimum Argo Events controller version or minimum Kubernetes version to users — especially relevant in BYO mode where the chart dependency is skipped.

## Sub-items

- [x] Add `annotations` block to `Chart.yaml`:
  ```yaml
  annotations:
    artifacthub.io/signKey: ""
    artifacthub.io/changes: |
      - Initial release
    sympozium.ai/argoEventsMinVersion: "1.7.0"
    sympozium.ai/kubernetesMinVersion: "1.25.0"
  ```

- [x] Add a `kubeVersion` field to `Chart.yaml` (Helm built-in):
  ```yaml
  kubeVersion: ">=1.25.0"
  ```
  Helm will reject the install with a clear error on unsupported clusters.

- [x] Add a comment in `values.yaml` under `argoEvents.install: false` (BYO mode) noting the controller version requirement:
  ```yaml
  # BYO mode: requires Argo Events controller >= 1.7.0
  # dataTemplate parameter syntax and conditionsReset.byTime require >= 1.7.
  ```

- [x] Confirm the Argo Events Helm chart version `2.4.x` maps to controller ≥ 1.9 — cross-reference the argo-events Helm chart changelog to record the exact chart→controller version mapping in a comment in `Chart.yaml`.
  - **Finding**: 2.4.x straddles the boundary. Chart 2.4.0 → v1.8.0, chart 2.4.1 → v1.8.1; chart 2.4.2+ → v1.9.x (min safe version for controller ≥ 1.9 is 2.4.2). Recorded as a comment in the `annotations` block in `Chart.yaml`.
