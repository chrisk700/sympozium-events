Replace the empty Helm scaffold `NOTES.txt` with post-install guidance. This is the first thing a user sees after `helm install`.

## Sub-items

- [x] Write `charts/sympozium-events/templates/NOTES.txt` with the following sections:

  **1. Quick health check**
  Commands to verify EventSource and Sensor reconciliation immediately after install:
  ```
  kubectl get eventsource,sensor -n {{ .Values.argoEvents.resourceNamespace }}
  ```
  Expected: both resources show `Ready: true`.

  **2. EventBus status**
  ```
  kubectl get eventbus -n {{ .Values.argoEvents.resourceNamespace }}
  ```
  The JetStream StatefulSet takes ~30s to reach `Running`. If EventSource/Sensor stay `NotReady`, check EventBus first.

  **3. No routes configured**
  If `.Values.routes` is empty, show a notice that no EventSource or Sensor resources were created and link to the values quickstart example.

  **4. Watching for AgentRun creation**
  ```
  kubectl get agentruns -n {{ .Values.namespace }} -w -l sympozium.ai/trigger=argo-events
  ```

  **7. Finding your instanceRef and agentId** (P14 finding — must be included)
  Explain that `instanceRef` is the name of a `SympoziumInstance` CR and show the discovery command:
  ```
  kubectl get sympoziuminstances -n sympozium-system
  ```
  Note that `agentRun.agentId` must match an agent configured in that SympoziumInstance. `event-triggered` is the convention used by the chart's own tests; users must set it to match their Sympozium deployment.

  **5. BYO mode note** (conditional on `argoEvents.install: false`)
  Remind user that Argo Events was not installed by this chart — the existing installation must be running and have an EventBus in `{{ .Values.argoEvents.resourceNamespace }}` named `{{ .Values.eventbus.name }}`.

  **6. Tailscale note** (conditional on `tailscale.enabled: true`)
  Remind user that the Tailscale Operator must be installed out of band. Show the command to check that annotations appeared on the EventSource Service.
