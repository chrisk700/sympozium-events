# P03 — Sympozium label strategy

The Sympozium CLI (current or future) should be able to list all active sympozium-events triggers across namespaces with a single label selector. The AgentRun already carries `sympozium.ai/trigger: argo-events`. The Sensor and EventSource need a matching label on the infrastructure side.

Current state: Sensor and EventSource carry `sympozium-events/route: <name>` and the standard Helm labels (`app.kubernetes.io/name`, `helm.sh/chart`, etc.) but no `sympozium.ai/*` label.

## Sub-items

- [x] Add `sympozium.ai/component: events-bridge` to the `sympozium-events.labels` helper in `charts/sympozium-events/templates/_helpers.tpl`. This propagates automatically to all resources that call `include "sympozium-events.labels"`: Sensor, EventSource, EventBus, RBAC, NetworkPolicy, tailscale Job.

- [x] Verify the AgentRun resource block in `sensor.yaml` carries both:
  - `sympozium.ai/trigger: argo-events` (already present)
  - `sympozium-events/route: <route.name>` (already present)
  No changes needed there — confirm by reading the template.

- [x] Document the selector in `polish.md` Decisions and in the README (P12):
  ```bash
  # List all active EventSources and Sensors managed by sympozium-events
  kubectl get eventsources,sensors -A -l sympozium.ai/component=events-bridge

  # List all AgentRuns created by event triggers
  kubectl get agentruns -n sympozium-system -l sympozium.ai/trigger=argo-events
  ```

- [x] Run `helm template sympozium-events ./charts/sympozium-events -f charts/sympozium-events/ci/two-routes.yaml` and confirm `sympozium.ai/component: events-bridge` appears on all emitted resources.
