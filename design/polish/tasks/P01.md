# P01 — PRD: close resolved open questions

Update `docs/prd/prd.md` to reflect findings from T02–T17. The PRD was written before implementation; several open questions were answered and the design changed.

## Sub-items

- [x] **Open Questions section**: mark each question as closed with its outcome
  - OQ-1 EventBus reuse → FAILS. `jetstream` type always owns its StatefulSet. `exotic` external mode is legacy `nats` type only. Argo Events runs a second bus (~300Mi idle, fully isolated).
  - OQ-2 conditionWindow scope → NOT a native Argo Events field. Implemented as `conditionsReset.byTime.cron` — a wall-clock cron schedule. Per-Sensor (not global). Coarser than a rolling TTL. Known reliability gap: does not fire if Sensor pod is down (argo-events#1543).
  - OQ-3 Argo Events version floor → confirmed ≥ 1.7 for `dataTemplate`; ≥ 1.6 for `conditionsReset.byTime`. Chart locks `2.4.x` (Helm) / controller ≥ 1.9.
  - OQ-4 sessionKey uniqueness → Sympozium does NOT enforce uniqueness. Duplicate AgentRuns are created. Argo Events deduplicates by `Nats-Msg-Id` header only — not by payload content. Two identical POSTs create two AgentRuns.
  - OQ-5 Cleanup policy → `spec.cleanup` field existence on AgentRun CRD unconfirmed (see P10).

- [x] **Key Values section**: remove `reuseSympoziumEventbus: true` example key and its comment — the field was never implemented; the PRD design was superseded by the T02 finding.

- [x] **Decisions section**: add entries for:
  - EventBus reuse not supported; Argo Events always manages its own StatefulSet.
  - `conditionWindow` renamed `conditionsResetCron` in the chart; semantics differ from the PRD concept.
  - Sympozium NATS service name is `nats` (not `eventbus-default-js` as assumed).

- [x] **Implementation Notes section**: update `conditionWindow scope` note to reflect the `conditionsReset.byTime.cron` implementation and its known reliability gap.

- [x] **Architecture diagram**: update the EventBus line to reflect that Argo Events runs its own JetStream — remove the "(Sympozium's NATS JetStream — shared, pending validation)" qualifier.
