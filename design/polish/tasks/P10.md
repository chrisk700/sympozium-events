# P10 — spec.cleanup field

The PRD mentions `spec.cleanup: delete` (default) vs `keep` (debugging) on the AgentRun CR. It's not in `values.yaml` or `sensor.yaml`. Before adding it, confirm whether the field actually exists on the Sympozium `AgentRun` CRD.

**Requires**: live Sympozium cluster with CRD installed.

## Sub-items

- [x] Inspect the AgentRun CRD schema:
  ```bash
  kubectl get crd agentruns.sympozium.ai -o jsonpath='{.spec.versions[0].schema.openAPIV3Schema.properties.spec}' | jq .
  ```
  Look for a `cleanup` field.

- [x] If the field exists:
  - Add `agentRun.cleanup: delete` to `values.yaml` (default `delete`; set `keep` to preserve completed runs for debugging Sensor parameter mapping).
  - Emit `cleanup: {{ $.Values.agentRun.cleanup }}` in the AgentRun resource block in `sensor.yaml`.
  - Document in `values.yaml` comment: `# keep — AgentRun CR is retained after completion for debugging. delete — CR is deleted after the run finishes.`

- [x] If the field does not exist:
  - Remove the reference from the PRD open questions section (P01).
  - Note it as a future Sympozium feature request.

- [x] Update P01 (PRD OQ-5) with the finding regardless of outcome.

## Finding

`cleanup` field confirmed in AgentRun CRD: `enum: [delete, keep]`, CRD default `delete`. Description: "Cleanup policy: delete to remove pod after completion, keep for debugging."
