## Results

**Option 1 works.**

The correct format is a **top-level `spec.service` field** on the EventSource (NOT nested inside the webhook source map):

```yaml
spec:
  service:
    metadata:
      annotations:
        tailscale.com/expose: "true"
        tailscale.com/hostname: sympozium-events-<name>
    ports:
      - port: 12002
        targetPort: 12002
  webhook:
    my-source:
      ...
```

`service.metadata.annotations` passes through verbatim to the controller-managed Service. Verified on Argo Events v1.9.10 — both Tailscale annotations appeared on the created Service.

**Options 2 (post-install Job) and 3 (Kyverno) are not needed for v1.** T11 uses annotation injection via `spec.service.metadata.annotations`. T12 ships as a documented fallback (gated on `annotationMode: job`); Kyverno is not implemented.
