# T02 — EventBus reuse validation

Critical question: can Argo Events connect to Sympozium's existing NATS JetStream, or does it need its own StatefulSet?

## Sub-items

- [x] Install Argo Events in the cluster: `helm install argo-events argo-events/argo-events -n argo-events --create-namespace`
- [x] Create a minimal Argo Events `EventBus` CR pointing at Sympozium's NATS (`nats://eventbus-default-js.sympozium-system:4222`)
- [x] Create a trivial `EventSource` (webhook) and `Sensor` with a log-only trigger
- [x] Fire a test event — confirm the Argo Events controller receives it without errors
- [x] Check Sympozium NATS for degradation (message loss, reconnect churn) during the test
- [x] **Decision gate**

## Results

**RESULT: Reuse FAILS.**

The Argo Events `jetstream` EventBus type always manages its own StatefulSet — the `exotic` external connection mode is only available for the legacy non-JetStream `nats` EventBus type.

- Default flipped to `false`. Argo Events runs its own JetStream (3-pod StatefulSet, ~300Mi idle).
- No degradation to Sympozium NATS observed (buses are fully isolated).
- Sympozium NATS service name is `nats` (not `eventbus-default-js` as assumed in PRD).
