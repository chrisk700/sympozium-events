# T07 — Sensor template

`templates/sensor.yaml` — the main generator, one Sensor per route.

## Sub-items

- [x] One `dependency` per source in `routes[].sources`; `condition` field from `routes[].condition` if present
- [x] Per target in `routes[].targets`: one trigger block
  - `k8s` trigger creating `AgentRun` with `generateName: <route.name>-`
  - `parameters`: `sessionKey` from source-type-specific default event ID path; `task` from `taskTemplate`
  - `filters` block from `targets[].filters` passed through verbatim — no abstraction
- [x] `conditionWindow` from `routes[].conditionWindow`, falling back to global `.Values.conditionWindow`
- [x] `helm template` against an example values file with two routes; verify the output is a valid `Sensor` spec

## Cross-document updates

No direct PRD gap from T07 itself. Note: the `conditionWindow` key implemented here was renamed to `conditionsResetCron` in T16 after live validation revealed Argo Events has no native per-event rolling window. PRD updates for this finding are attributed to T16. See T16 cross-document section.
