# P15 — Endpoint auth: secure by default

Webhook sources are exposed services on the cluster (tailnet or public internet). Auth should be on by default with a simple opt-out.

## Design (final — after redesign)

Auth is **per webhook source**, inside the source definition. No global auth block, no chart-generated Secrets. Users bring their own Secret.

| `sources[].auth.secretName` | `sources[].auth.allowInsecure` | Behaviour |
|---|---|---|
| `"my-secret"` | `false` (default) | HMAC enabled — chart wires Secret into Argo Events `authSecret` |
| any / absent | `true` | HMAC disabled — no `authSecret` emitted; NOTES.txt warns |
| absent | `false` | Template error — missing auth is not a silent insecure default |

HMAC mechanism: Argo Events native `authSecret` field per webhook source. Validates `X-Hub-Signature-256: sha256=<hmac>`. Compatible with GitHub, PagerDuty, and any sender that supports webhook secrets.

Non-webhook sources (sqs, sns, calendar, resource, nats) carry their own auth in the source definition (`credentialsSecret` etc.) and are unaffected.

## Redesign note

The initial implementation used a top-level `auth:` block (global) with per-route override. This was wrong: it promoted webhooks above other source types, `allowInsecure` was chart-global (all-or-nothing), and it conflicted with how every other source type handles auth (per-source, in the source definition). The redesign moves auth entirely into the source definition, removes chart-generated Secrets, and enforces explicit configuration per webhook source.
