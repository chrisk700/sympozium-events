# T03 — AgentRun k8s trigger validation

Prove the core pattern: Sensor `k8s` trigger creates an `AgentRun` CR that Sympozium executes.

## Sub-items

- [x] Apply the cross-namespace RBAC from the PRD (`Role` in `sympozium-system`, `RoleBinding` binding the `argo-events` SA)
- [x] Hand-author a Sensor with a webhook `EventSource` and a `k8s` trigger producing a minimal `AgentRun`
- [x] Use `generateName` on the CR; derive `sessionKey` from a stable field in the test payload
- [x] POST a test webhook — confirm `AgentRun` CR appears in `sympozium-system`
- [x] Confirm Sympozium controller reconciles it and the run reaches `Succeeded`
- [x] Confirm: does Sympozium enforce `sessionKey` uniqueness, or silently tolerate duplicates?
- [x] Confirm: Argo Events version floor for `dataTemplate` syntax — installed version must be ≥ 1.7

## Results

**sessionKey uniqueness**: Sympozium silently tolerates duplicate sessionKeys. Two AgentRuns with `sessionKey=t03-test-001` were created and both reconciled independently. No rejection, no error. Deduplication must be handled at the Sensor level (Argo Events circuit breaker / event deduplication) — not by Sympozium.

**dataTemplate version floor**: Installed v1.9.10. `dataTemplate` used in live test — both `spec.sessionKey` and `spec.task` correctly templated from webhook payload. Version floor ≥ 1.7 confirmed.
