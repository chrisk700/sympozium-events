# P08 — Tailscale Funnel end-to-end

T04 confirmed that EventSource `spec.service.metadata.annotations` passes through to the controller-managed Service. T11 implemented the chart support. What was not validated: an actual Tailscale Funnel connection from outside the cluster reaching the EventSource and producing an AgentRun.

**Requires**: live cluster with Tailscale Operator installed out of band.

## Sub-items

- [x] Install Tailscale Operator in the cluster (out of band — operator install docs, not chart responsibility).

- [x] Deploy `sympozium-events` with Tailscale annotations enabled:
  ```yaml
  tailscale:
    enabled: true
    annotationMode: annotation
    annotations:
      tailscale.com/expose: "true"
      tailscale.com/hostname: sympozium-events-test
  ```
  Confirm the EventSource Service receives both annotations (`kubectl get svc -n argo-events -o yaml`).

- [x] Confirm Tailscale Operator reconciles the Service and assigns a `*.ts.net` hostname. Check `tailscale status` or the Tailscale admin console.
  **Finding**: hostname `sympozium-events-test.barred-porgy.ts.net` assigned within ~45s. Proxy pod `ts-smoke-test-eventsource-svc-vrwx2-0` running in `tailscale` namespace. The operator picked up the `tailscale.com/expose` annotation on the ClusterIP Service created by Argo Events without issue.

- [x] Enable Funnel on the hostname (if not automatic from annotations — check whether `tailscale.com/funnel: "true"` is the correct annotation or if it requires a separate `tailscale funnel` CLI command).
  **Finding**: `tailscale.com/funnel: "true"` on a Service annotation is silently ignored by the operator — Funnel only works via a Kubernetes Ingress with `ingressClassName: tailscale`. Added `tailscale-ingress.yaml` template to the chart (creates one Ingress per webhook route when `tailscale.funnel: true`). Funnel also requires an ACL `nodeAttrs` entry granting the `funnel` attribute to `tag:k8s` — without it the proxy pod starts but Funnel is inactive. Tailnet-only mode (Service annotation) confirmed working.

- [x] POST a test webhook payload to the `*.ts.net` URL from outside the cluster (e.g. from a laptop not on the same tailnet):
  ```bash
  curl -X POST https://sympozium-events-test.<tailnet>.ts.net/<webhook-endpoint> \
    -H 'Content-Type: application/json' \
    -d '{"test": true}'
  ```
  Confirm an `AgentRun` is created in `sympozium-system`.
  **Result (tailnet)**: `curl -X POST http://sympozium-events-test.barred-porgy.ts.net:12000/smoke-hook` → HTTP 200. AgentRun `smoke-test-294sw` created in `sympozium-system`, status `Succeeded` within 10s. Full pipeline confirmed: tailnet → EventSource → Sensor → AgentRun → Sympozium.
  **Funnel (public internet)**: not validated — requires ACL `nodeAttrs` funnel grant and Ingress-based deployment. Chart now supports this via `tailscale.funnel: true`.

- [x] Document the required `tailscale.annotations` values for Funnel mode in `values.yaml` comments and the README.
  **Done**: Added `## Tailscale` section to README covering both modes, ACL prerequisite, and per-route hostname override. Updated values table rows for `tailscale.funnel`, `tailscale.annotations`, `tailscale.annotationMode`. Added Known Limitation for Funnel not validated. Updated values.yaml block comment to describe both modes clearly.

- [x] Record findings here and close the Tailscale open question in the PRD.
  **Done**: PRD Security section updated with both exposure modes and Funnel Ingress finding. PRD Workstreams section updated with validation status. PRD Implementation Notes `Tailscale` entry rewritten to cover both modes. PRD Open Questions: Tailscale added to Closed (partially) — tailnet validated, Funnel pending ACL.

## Failure modes checked

- **Operator reconciles third-party Services** ✓ — Tailscale Operator reconciled the Argo Events-managed ClusterIP Service without issue. Proxy pod created within ~45s.
- **Funnel requires Ingress, not Service annotation** ✓ — `tailscale.com/funnel: "true"` on a Service is silently ignored. Chart updated to create Ingress resources for Funnel mode.
